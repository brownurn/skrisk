# SK Risk V1 Implementation Plan

> **Context:** This plan was created from the initial SK Risk design and then updated to reflect the first implemented repository state.

**Goal:** Build a `cwsrisk`-style intelligence platform for AI agent skills, starting with `skills.sh` discovery, linked GitHub repos, repeated rescans, and a public API/dashboard.

**Architecture:** SK Risk is a standalone Python service with async storage, registry collectors, Git-backed repo loading, heuristic risk analysis, and a FastAPI UI/API surface. The design follows the existing Melurna scanner pattern: SK Risk owns collection and analysis, shared services such as `mewhois` and `meip` are integrated later, and downstream products can consume the feed.

**Tech Stack:** Python 3.12, FastAPI, SQLAlchemy asyncio, SQLite bootstrap path, Click CLI, APScheduler, Jinja templates.

---

## Implemented In This Repository

- `skills.sh` sitemap and audits parsing
- GitHub-backed skill discovery and local file loading
- Plugin-manifest discovery for Claude plugin marketplace layouts
- Repeated scan observations with repo snapshots and skill snapshots
- Heuristic analyzer for prompt injection, remote execution, exfiltration, and Base64-obfuscated payloads
- Registry sync orchestration with per-entry failure isolation
- FastAPI JSON API and HTML dashboard
- CLI commands for DB initialization, directory setup, registry sync, scheduling preview, and serving the app
- Regression coverage for rescans, mirror refresh, discovery breadth, sync isolation, API, dashboard, and CLI

## Locked Product Decisions

- V1 anchor: `collection-first`
- V1 discovery boundary: `skills.sh` plus linked GitHub repositories
- Default rescan cadence: every 72 hours
- Public-facing SK Risk dashboard/API ships in this repo; Melurna integration is downstream
- `cwsrisk` is the reference scanner pattern; `mewhois` and `meip` are reused later instead of duplicated

## Remaining Planned Work

### Collection and Storage

- Replace assumed GitHub source URLs with source URLs resolved directly from registry metadata or repo pages
- Add persistent scan-run tables and scan-event history distinct from content/artifact identity
- Roll the bootstrap SQLite path onto the production Postgres/Timescale storage design

### Intelligence and Enrichment

- Integrate `mewhois` and `meip` for domain/IP enrichment
- Add external vendor adapters in priority order: VirusTotal, URLhaus, AbuseIPDB, OTX, and passive DNS later if needed
- Expand analyzer coverage for dependency abuse, browser/session theft, and infrastructure rarity scoring

### Product and Operations

- Schedule background sync execution on a live 72-hour loop
- Add richer skill detail views, version history, IOC drilldowns, and repo/publisher rollups
- Add Melurna feed/export support once the source schema stabilizes
